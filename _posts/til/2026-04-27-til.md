# Today I Learned (2026-04-27)

## 청구서 수동 발행 페이지

- 청구서 수동 발행 페이지 구현을 진행했다.
- 입력 폼, 미리보기, PDF 생성 흐름이 서로 연결되기 때문에 데이터 구조를 먼저 안정적으로 잡는 것이 중요했다.
- 실제 데이터 연동 전에도 fixture와 스텁을 활용하면 UI와 사용 흐름을 빠르게 검증할 수 있다.

## 코드 정리

- 입력 폼, 미리보기, 다운로드 액션을 한 컴포넌트에 몰아넣지 않고 역할별로 분리했다.
- 청구서 데이터 타입을 기준으로 폼 값과 PDF 값의 차이를 확인하며 변환 지점을 명확히 잡았다.
- fixture를 프리셋 형태로 관리하면 여러 케이스를 빠르게 바꿔가며 화면을 확인할 수 있다.
