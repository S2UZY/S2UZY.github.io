# Today I Learned (2026-02-25)

## 모노레포 + Tailwind 우선순위 문제

- `@repo/ui`에서 Tailwind를 CSS로 변환한 파일을 읽는 구조 때문에 스타일 우선순위가 꼬였다.
- 일반 `.css` 파일 우선순위가 강해져 의도와 다른 결과가 발생했다.
- `@repo/ui` 쪽에 `@layer base`를 지정해 조정해봤고, 일부 프로젝트(어드민)는 또 다른 이슈가 생겨 케이스별 대응이 필요했다.
- 폰트가 포함되면서 CSS 파일 크기가 커졌고, `@layer base` 적용 시 버벅임이 심해져 `next/font/local`로 개별 사용하도록 옮겼다.
- 당장 해결은 했지만, 장기적으로는 `@repo/ui`를 Next.js 프로젝트에서 직접 스캔하도록 구조 개선을 검토할 필요가 있다.
